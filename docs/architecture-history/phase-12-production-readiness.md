# Phase 12 — Production Readiness

**Status**: ✅ Complete

## Summary

**Part A — Outbox pattern** (Steps 1–7): `domain/outbox/models.py` — `OutboxEntry` (tenant_id, topic, key, payload, retry_count, published_at). `domain/outbox/repository.py` — Protocol with `insert(session=)`, `find_pending`, `mark_published`, `increment_retry`. `infrastructure/mongo/repositories/outbox_repository.py` — `MongoOutboxRepository` whose `find_pending` is deliberately cross-tenant (relay processes all tenants). `services/outbox/relay.py` — `OutboxRelay` with 500ms poll loop, batch-100 tick, JSON-serialized Kafka publish, `increment_retry` on failure. `orchestration/workers/outbox_relay_worker.py` — does NOT extend `BaseConsumer` (it's a MongoDB poller, not a Kafka consumer); owns its own Kafka producer + OTel + MetricsServer. `JobService` rewritten: `create_job` wraps both `jobs.insert` and `outbox.insert` in a Mongo transaction via `client.start_session()` + `session.start_transaction()`. Direct `KafkaProducer.publish` from `JobService` is now forbidden. `domain/jobs/repository.py` and `infrastructure/mongo/repositories/job_repository.py` updated with `session: object | None = None` in `insert` (opaque parameter — infrastructure-agnostic). `docker/outbox_relay.Dockerfile` (base deps only). `docker-compose.app.yml` — added `outbox-relay-worker` service. Tests: `tests/integration/test_outbox_relay.py` (5 tests: atomic write, relay publish, Kafka failure, idempotency).

**Part B — Real authentication** (Steps 8–17): `domain/auth/models.py` — `Tenant`, `ApiKey` (hash-only, key_prefix display), `AuthContext`. `domain/auth/repository.py` — Protocol for `TenantRepository` and `ApiKeyRepository`. `infrastructure/auth/api_key.py` — `generate_api_key(env)`, `hash_key`, `key_prefix`, `parse_env` (pure functions). `infrastructure/auth/jwt.py` — `JWTValidator` with lazy JWKS fetch, multiple IdP support, `https://trendstorm.ai/tenant_id` custom claim. `infrastructure/mongo/repositories/tenant_repository.py` — `MongoTenantRepository` (does NOT extend `TenantScopedRepository` — tenants are root entities). `infrastructure/mongo/repositories/api_key_repository.py` — `MongoApiKeyRepository` with hot-path `get_by_hash`. `shared/config` — `AuthMode(StrEnum)` (disabled/header/key/oauth/key_or_oauth), `AuthSettings` with rate-limit params. `services/auth/service.py` — `AuthService`: `authenticate_by_key`, `authenticate_by_jwt`, `create_key`, `revoke_key`, `rotate_key`, `list_keys`. `shared/rate_limit/redis_bucket.py` — generic `RedisBucket` with same Lua script as Scout (key is a plain string). `api/middleware/auth.py` — `AuthMiddleware` (refuses `disabled` + `ENV=prod` at init; reads `auth_service` lazily from `app.state`; sets `request.state.auth_context` + `request.state.tenant_id`). `api/middleware/rate_limit.py` — `RateLimitMiddleware` (lazy-init `RedisBucket` from `app.state.redis` on first request; returns 429 with `Retry-After` + `X-RateLimit-Limit`). `api/middleware/tenant.py` — thin shim (no-op if auth_context already set). `api/main.py` — builds `AuthService` + `JWTValidator` in lifespan; middleware order: CORS → Tenant → RequestLogging → RateLimit → Auth → CorrelationId. `api/routers/api_keys.py` — POST `/v1/api-keys` (returns plaintext once), GET `/v1/api-keys`, DELETE `/v1/api-keys/{id}`. `api/deps.py` — `get_auth_service` + `AuthServiceDep`. `scripts/generate_api_key.py` — bootstrap CLI for first tenant + key. Tests: `tests/unit/test_auth.py` (20 tests: key generation, hashing, prefix, AuthService key + JWT paths, ApiKey model properties).

**Part C — Cost governance** (Steps 18–24): `domain/billing/models.py` — `CostLedgerEntry` (immutable, append-only), `TenantQuotas`, `QuotaStatus`, `PLAN_QUOTAS` (free: $10/50 jobs, pro: $100/500 jobs, enterprise: $1000/10k jobs). `domain/billing/repository.py` — `CostLedgerRepository` Protocol. `infrastructure/mongo/schema.py` — `COST_LEDGER` added. `infrastructure/mongo/indexes.py` — 3 cost ledger indexes (tenant+created, tenant+job, TTL 90 days). `infrastructure/mongo/repositories/cost_ledger_repository.py` — `MongoCostLedgerRepository` with `monthly_spend_usd_micro` and `jobs_this_month` aggregation pipelines. `services/billing/prices.py` — `_PRICE_TABLE` (micro-dollars/1k tokens per provider+model); `compute_cost_usd_micro()` with cached-token 10% discount. `services/billing/quota_service.py` — `QuotaService.check(tenant_id)` → `QuotaStatus`; parallel ledger reads; plan-based thresholds from `PLAN_QUOTAS`. `JobService` updated with optional `quota: QuotaService` — pre-flight check raises `BusinessRuleError(code="quota_exceeded")` → HTTP 402. `shared/metrics/cost.py` — `record_llm_cost()` now accepts `job_id` + `ledger`; fire-and-forget `loop.create_task(_write())` for ledger insert. `services/analysis/analyst.py` — `produce_analysis` and `_generate_analysis` accept `ledger` param, thread to `record_llm_cost`. `orchestration/workers/analyst_worker.py` — `MongoCostLedgerRepository` wired in; passed to `produce_analysis`. `api/routers/api_keys.py`, `api/routers/quota.py` — GET `/v1/quota` endpoint. `api/error_handlers.py` — `BusinessRuleError(code="quota_exceeded")` → HTTP 402. Tests: `tests/unit/test_billing.py` (13 tests: price math, quota service, ledger entry model). `tests/unit/test_auth.py` (20 tests).
