# Phase 9 — Streaming + Publisher Agent

**Status**: ✅ Complete

## Summary

`services/publish/renderer.py` — `RenderEngine` renders Analysis to three formats. `render_markdown()` uses a Jinja2 template (injected via importlib.resources) with the full insights + citations structure. `render_json()` serializes the Analysis pydantic model to bytes. `render_pdf()` converts the rendered Markdown HTML via `weasyprint.HTML(string=html).write_pdf()` — best-effort; on macOS dev without GTK/Pango this fails silently; the service catches and logs. `services/publish/service.py` — `PublisherService.publish(analysis, category) → PublishResult`. Renders all three formats, uploads each to MinIO using `report_key(tenant_id, job_id, report_id, fmt=...)` path builder, inserts three `Report` documents to Mongo, returns `PublishResult(markdown_report_id, pdf_report_id | None, json_report_id)`.
`agents/publisher/pipeline.py` — `PublisherPipeline`: loads `Analysis` + `Category` from Mongo repos, delegates to `PublisherService`, returns `PublishPipelineResult`. Raises `NotFoundError` if either document is missing. Clean separation: pipeline owns I/O wiring (repos, config); service owns rendering logic.
`agents/orchestrator/nodes.py` — `publish_node` is now dual-mode (same pattern as ingest/embed/analyze): if `kafka_producer` in configurable → publishes `PublishPendingEvent(job_id, analysis_id, category_id)`, returns partial state (graph pauses via `interrupt_after=[NODE_PUBLISH]`); otherwise stub path advances directly to COMPLETED for unit tests.
`orchestration/topics.py` — added `PUBLISH_COMPLETED`, `RETRY_PUBLISH_30S`, `RETRY_PUBLISH_5M`, `RETRY_PUBLISH_1H`. `orchestration/events.py` — `PublishPendingEvent` gains `category_id`. New `PublishCompletedEvent(success, markdown_report_id, pdf_report_id, json_report_id, error_code, error_message)`. Both added to `AnyEvent` discriminated union.
`orchestration/workers/publisher_worker.py` — `PublisherWorker(BaseConsumer)`. Idempotency key: `f"publisher:{job_id}"`. On success: emits `StreamEvent(STAGE_STARTED)` before render, `StreamEvent(REPORT_READY)` after, publishes `PublishCompletedEvent(success=True)`. On `NotFoundError`: publishes `PublishCompletedEvent(success=False)` — permanent failure, no retry. Other exceptions re-raise for Kafka retry topology (`RETRY_PUBLISH_30S → 5M → 1H → DLQ`). Includes full `run_worker()` + `main()` entrypoints.
`orchestration/workers/sse_coordinator_worker.py` — `SSECoordinatorWorker(BaseConsumer)`. Consumes `stream.partial.v1`. Idempotency key: `f"sse:{event.event_id}"` so duplicate Kafka deliveries don't double-increment seq. `_process()`: `incr_seq(job_id)` → `stream_store.append(job_id, payload)` → `pubsub.publish(job_id, payload)`. Unknown `StreamEventType` values are logged (best-effort) but still stored. Includes full `run_worker()` + `main()` entrypoints. SSE events are ephemeral UX signals — failures go to DLQ immediately rather than the tiered retry topics (retry delay makes stale stream events worse, not better).
`orchestration/workers/orchestrator_worker.py` — now subscribes to FIVE topics (`PUBLISH_COMPLETED` added). `_resume_to_publish` (renamed from `_resume_to_completion`) injects `kafka_producer` into configurable and uses `interrupt_after=[NODE_PUBLISH]`. New `_handle_publish_completed`: on success → `aupdate_state(stage=COMPLETED, publishing=PublishingState(...))` as_node=NODE_PUBLISH, `astream(None)` → END, then `update_status(COMPLETED)`; on failure → `update_status(FAILED)` without resuming graph.
`services/streaming/emit.py` — `emit_stream_event(StreamEvent, *, producer, correlation_id)` publishes a `StreamPartialEvent` to `stream.partial.v1` via the Kafka producer. Used by `PublisherWorker` (and future streaming agents) as a thin helper to avoid repeating the serialization pattern.
`domain/streaming/events.py` — `StreamEventType` StrEnum with `STAGE_STARTED`, `CHUNK_DELTA`, `REPORT_READY`, `JOB_FAILED`. `is_terminal` property on `StreamEventType` (True for `REPORT_READY` / `JOB_FAILED`). `StreamEvent` Pydantic model.
`services/streaming/sse.py` — `sse_event_generator()`: subscribe-before-read pattern (SUBSCRIBE Pub/Sub first → XRANGE replay → tail live Pub/Sub). De-duplication via `seen_seqs` set. Heartbeat comments on `settings.heartbeat_seconds` interval. Generator closes on terminal event (`is_terminal` check). Wires into `GET /v1/jobs/{job_id}/stream` via `EventSourceResponse`.
`infrastructure/redis/streams.py` — `RedisStreamStore`: `append()` (XADD + refresh TTL), `incr_seq()` (INCR with co-expire), `read_from(min_seq=N)` (XRANGE with seq filter for Last-Event-ID resumption). `infrastructure/redis/pubsub.py` — `RedisPubSub`: `publish()`, `subscribe()` → `_PubSubIterator` with `stop_event` support (races listener vs event via `asyncio.wait`). Both classes are DI-injected with the live Redis client via `.init(redis_client)` — no lifecycle ownership.
`docker/publisher.Dockerfile` — multi-stage, `publish + blob` dep groups + weasyprint system libs (`fonts-dejavu`, `libpango-1.0-0`, `libpangoft2-1.0-0`, `libcairo2`, `libharfbuzz0b`, `libglib2.0-0`). `docker/sse_coordinator.Dockerfile` — base deps only (smallest image in the fleet — no LLM SDKs, no blob, no vector store). `docker-compose.app.yml` — adds `publisher-worker` + `sse-coordinator-worker` services.
**Tests**: 824 unit tests. `tests/unit/test_publisher.py` (40 tests) covers `RenderEngine` (PDF mocked via `patch.dict(sys.modules, {"weasyprint": ...})`) and `PublisherService` (renderer mocked via `MagicMock(spec=RenderEngine)` — never touches weasyprint on macOS dev). `test_publisher_pipeline.py` (10 tests). `test_publisher_worker.py` (14 tests). `test_orchestrator_analysis_handler.py` extended with `TestPublishCompletedHandler` (7 tests) + subscription coverage check. `tests/integration/test_publisher_e2e.py` (2 tests — happy path + permanent failure). `tests/integration/test_sse_endpoint.py` (3 tests — history replay, terminal close, 404). `tests/integration/test_full_pipeline_with_sse.py` (1 `@slow` test — full Phase 9 pipeline: publisher + SSE coordinator with real Redis Streams + Pub/Sub verification).
