# Phase 7 — Knowledge Agent (Chunking + Embedding Pipeline)

**Status**: ✅ Complete

## Summary

`domain/llm/` — new module. `providers.py`: two separate `@runtime_checkable` Protocols — `EmbeddingProvider` (`model_id`, `dimensions`, `max_batch_size`, `max_input_tokens`, `embed_batch`) and `ChatProvider` (`model_id`, `complete`, `stream`). Never combined. `models.py`: `Message`, `Completion`, `EmbeddingBatchResult`. `domain/vectors/`: `VectorStore` Protocol (`health_check/upsert/query/delete_by_filter`) and `VectorHit` model.
`infrastructure/llm/` — concrete providers. `gemini.py`: `GeminiEmbeddingProvider` wraps `google-genai` sync client via `asyncio.to_thread`; uses `google.genai.types.EmbedContentConfig`. `openai.py`: `OpenAIEmbeddingProvider` wraps native `AsyncOpenAI` directly. `ollama.py`: `OllamaEmbeddingProvider` via `ollama.AsyncClient.embed()`. `retry.py`: `RetryingEmbeddingProvider` with full-jitter exponential backoff (`uniform(0, min(max_delay, base*2^attempt))`); catches `LLMTransientError` only (parent of `LLMRateLimitError`/`LLMTimeoutError`); injectable `_sleep` and `_random_fn` for unit tests. `registry.py`: `build_embedding_provider(settings) → EmbeddingProvider` selects provider by `settings.llm.default_embedding_provider`. `anthropic.py`: stub `AnthropicChatProvider` that raises `NotImplementedError("...Phase 8...")`.
`shared/errors` — LLM error hierarchy extended: `LLMTransientError` and `LLMPermanentError` and `LLMSchemaError` added as children of `LLMError`; `LLMRateLimitError`/`LLMTimeoutError` re-homed under `LLMTransientError` (not directly under `LLMError`). `shared/config` — `LLMProvider.GEMINI` added, `GeminiSettings` with `api_key/embedding_model/chat_model`, `LLMSettings` gains `default_embedding_provider`, `default_chat_provider`, `retry_max_attempts`, `retry_base_delay_seconds`, `retry_max_delay_seconds`.
`infrastructure/vectors/chroma_store.py` — `ChromaVectorStore`: `deferred import chromadb` in `connect()` (avoids onnxruntime import at module load), `_coll_cache`, cosine-space collections via `get_or_create_collection(name, metadata={"hnsw:space": "cosine"})`. Score conversion: `max(0, min(1, 1 - distance/2))` maps Chroma's [0,2] cosine distance to [0,1]. Collection naming: `f"chunks__{tenant_id[:8].lower()}__{model_id.replace('.','_').replace('-','_')}"` (per-tenant per-model isolation).
`infrastructure/blob/minio_client.py` — added `download(bucket, key) → bytes` via `get_object`.
`agents/knowledge/` — new module. `tokenizer.py`: `count_tokens`/`truncate_to_tokens` using `tiktoken` `cl100k_base` encoding universally. `chunker.py`: `ParentChildChunker` — pysbd `Segmenter(language="en", clean=False)` for sentence segmentation with cursor tracking for char offsets; `_build_windows` accumulates sentences until token budget exhausted (`break` when `end >= len(sentences)` prevents spurious tail parent); parent windows overlap by `parent_overlap_sentences` sentences. `RawChunk(frozen=True)` carries `text, token_count, position, parent_index, char_start, char_end`; `is_parent` property (`parent_index is None`). `pipeline.py`: `KnowledgePipeline.process_document()` — idempotency via `list_by_document(tenant_id, doc_id, embedding_model=model_id)` (checks whether child chunks for this document already exist under the current embedding model), downloads text from MinIO, chunks, `bulk_insert` to Mongo, embeds in `max_batch_size` batches, upserts vectors to ChromaDB, calls `set_vector_id` per child chunk. `KnowledgeResult(document_id, n_chunks_created, n_vectors_upserted, skipped)`. Idempotency scoped to `(tenant_id, document_id, embedding_model)` — switching embedding providers triggers re-embedding under the new model, with old chunks becoming orphans that TTL out after 1 year.
`orchestration/events.py` — `KnowledgePendingEvent` updated: `document_refs: list[KnowledgeDocRef]` (each carrying `document_id, blob_uri_text, category_id, source_id`). `KnowledgeCompletedEvent` updated: `document_results: list[KnowledgeDocResult]`, `failed_document_ids: list[str]`. `KnowledgeDocResult(document_id, n_chunks, n_vectors, skipped)`.
`orchestration/workers/knowledge_worker.py` — `KnowledgeWorker(BaseConsumer)`. Consumes `knowledge.pending.v1`. Idempotency key: `knowledge:{job_id}`. asyncio.Queue + N=4 workers (same producer-consumer pattern as ScoutWorker). Retry topology: `retry.knowledge.30s → 5m → 1h → dlq.v1`. On success publishes `KnowledgeCompletedEvent`.
`agents/orchestrator/nodes.py` — `embed_node` is now dual-mode (same pattern as `ingest_node`). Production path: builds `KnowledgeDocRef` list from `state.ingestion.raw_documents` using `_blob_uri_text()` helper (derives text URI from raw URI), publishes `KnowledgePendingEvent`, returns partial state (stage stays EMBEDDING). Stub path: synthetic chunks, advances to RETRIEVING.
`orchestration/workers/orchestrator_worker.py` — now subscribes to `[Topic.JOBS_REQUESTED, Topic.INGEST_COMPLETED, Topic.KNOWLEDGE_COMPLETED]`. `_handle_ingest_completed` now uses `interrupt_after=[NODE_EMBED]`. `_handle_knowledge_completed` builds `ChunkRef` list (one synthetic ref per non-skipped document, `new_id()` for ID — actual chunk IDs are in Mongo; this just signals embedding done), calls `aupdate_state(config, ..., as_node=NODE_EMBED)`, then `astream(None, config)`.
`docker/knowledge.Dockerfile` — multi-stage, `llm` + `rag` + `blob` dep groups (no `agents`/`ingest`). `docker-compose.app.yml` — added `knowledge-worker` with MinIO + ChromaDB env vars.
**pyproject.toml**: added `google-genai>=1.10.0,<2.0` (`llm` group), `pysbd>=0.3.4,<1.0` + `onnxruntime>=1.14.1,<1.20` (`rag` group — onnxruntime pinned for Intel Mac wheel availability). Added mypy ignores for `google.genai.*`, `pysbd.*`, `chromadb.*`, `tiktoken.*`. Added pysbd `SyntaxWarning` filter to pytest.
`api/main.py` — lifespan connects `ChromaVectorStore` alongside other clients; `build_embedding_provider` called with graceful warn if no key. `api/deps.py` + `api/routers/health.py` — `VectorStoreDep` wired; readiness check now 5-way gather including `vector_store`.
**Tests**: `tests/unit/test_chunker.py`, `test_knowledge_pipeline.py`, `test_llm_retry.py`, `test_llm_registry.py` — pure unit tests (injected `_sleep`/`_random_fn` for retry math). `tests/integration/test_chroma_store.py`, `test_knowledge_pipeline.py`, `test_knowledge_e2e.py` — require `make up`; e2e test resolves embedding provider (Ollama → Gemini fallback), uploads text to MinIO, publishes event, polls Mongo for chunks, verifies ChromaDB vectors, checks idempotency.
