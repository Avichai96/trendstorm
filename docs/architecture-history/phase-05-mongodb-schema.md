# Phase 5 — MongoDB Schema Deep Dive

**Status**: ✅ Complete

## Summary

`infrastructure/mongo/schema.py` — `Collection` StrEnum (canonical names; never string literals at call sites).
`infrastructure/mongo/indexes.py` — single source of truth for all indexes via `IndexSpec` dataclass. ~18 indexes: tenant-first compound indexes per collection, partial index on jobs for in-progress (derives non-terminal set from `JobStatus.is_terminal`), TTL indexes (jobs 90d, raw_documents/chunks/analyses/reports 1y, idempotency uses `expireAfterSeconds=0` with absolute `expires_at`), text index on `chunks.text` for BM25 hybrid (Phase 8), unique compound `(tenant_id, name)` on categories, unique `(tenant_id, url_hash)` on sources.
`infrastructure/mongo/repositories/_base.py` — `TenantScopedRepository[T]` mixin (NOT a generic CRUD base class). Subclass declares `collection: ClassVar[Collection]` and `model: ClassVar[type[T]]`. Provides `_tenant_query()` (the ONLY function in the codebase that constructs Mongo filters for tenant-scoped collections — defense in depth), `_encode/_decode`, `_insert/_find_one/_find_many` primitives with error mapping. Public method NAMES are still domain-meaningful (`list_pending_for_tenant`), not raw CRUD.
**Domain models created**: Category (name unique per tenant, keywords deduped case-insensitively, soft-archive), Source (URL hashed after canonicalization — lowercase host, sorted query, stripped tracking params, dropped fragment, dropped trailing slash, dropped default port; `url_hash` is unique per tenant), RawDocument (metadata only, raw text in MinIO via `blob_uri_raw/blob_uri_text`, `content_hash` for cross-source dedup), Chunk (text + vector_id pointer to ChromaDB; vectors NEVER in Mongo; supports parent-child chunking via `parent_chunk_id`), Analysis (Insight + Citation submodels, validator score/passed/notes, model provenance, refinement_loops), Report (per-format render, blob_uri to MinIO).
**Mongo repositories created**: MongoCategoryRepository (`find_one_and_update` with `return_document=True` for atomic partial update), MongoSourceRepository (`list_by_ids` preserves caller order via position map), MongoRawDocumentRepository (`find_by_content_hash` for dedup), MongoChunkRepository (`bulk_insert` with `ordered=False`, `get_many` preserves vector-score order), MongoAnalysisRepository, MongoReportRepository.
**Services**: CategoryService (explicit `get_by_name` before insert for nice errors; archive = soft delete), SourceService (MAX_SOURCES_PER_CATEGORY=500; verifies category exists; URL-validation errors caught as ValidationError).
**Routers**: `/v1/categories` (CRUD-ish; no DELETE, only archive) and `/v1/sources` (POST/GET/list-by-category). Both routers use `dependencies=[Depends(require_tenant)]` for OpenAPI/Swagger header documentation.
`MongoJobRepository.avg_duration_by_category` — example aggregation pipeline; `$match` MUST be first to hit the tenant-status-created index; pre-aggregation/`$merge` rollups deferred to Phase 11.
