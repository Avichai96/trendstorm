# Phase 6 — Scout Agent (Real Ingestion Pipeline)

**Status**: ✅ Complete

## Summary

`agents/scout/` — new module. `fetcher.py`: `Fetcher` class wraps `httpx.AsyncClient`; checks `Content-Type` against an allowlist, streams response up to `max_response_bytes`, strips MIME params for routing. `FetchResult` is a frozen dataclass carrying `raw_bytes`, `content_type`, `encoding`, `metadata` (status, final URL, elapsed). `hashing.py`: pure `content_hash(text: str) → str` (SHA-256 hex of UTF-8 bytes).
`agents/scout/rate_limit.py` — Redis token-bucket per `(tenant_id, host)`. A single atomic Lua script (`_RATE_LIMIT_SCRIPT`) performs the acquire + refill in one round-trip; returns `(allowed: bool, retry_after_ms: int)`. Keys: `scout:rl:{tenant_id}:{host}`.
`agents/scout/parsers/` — content-type-routed parsers. `html.py`: `trafilatura.extract` with BS4 fallback. `rss.py`: `feedparser` + BS4 HTML-strip for entry text. `api.py`: JSON stdlib. `sitemap.py`: `defusedxml.ElementTree` (prevents XXE/entity expansion); returns `discovered_urls` for queue expansion. `__init__.py` exposes `ParseResult` (frozen dataclass) and `route(FetchResult) → ParseResult`; generic XML peeks at root tag to distinguish sitemap from RSS.
`agents/scout/pipeline.py` — `ingest_sources()` top-level coroutine. Uses an `asyncio.Queue` + N worker tasks (producer-consumer) rather than bare `asyncio.gather` — avoids the thundering-herd / rate-limit problem identified in Phase 5. Concurrency default: 16 (`IngestSettings.concurrency_per_job`). Sitemaps discovered during processing push new `_FetchTask` items back onto the queue; `sitemap_source_ids` tracks which are parent sitemaps to avoid spurious `last_fetch_status` writes on sub-URLs. Dedup path: `find_by_content_hash` → if hit, return `SourceOutcome(status="deduplicated", document_ref=existing_ref)`. `ConflictError` on insert treated as a dedup race win. `IngestionResult` aggregates `SourceOutcome` items with computed properties `document_refs`, `failed_source_ids`, `deduped_count`.
`infrastructure/blob/minio_client.py` — `MinioClient` lifecycle wrapper using `aioboto3` (NOT the minio-py SDK — aioboto3 is already in the dep tree for S3-compatible stores). `connect()` creates the session, enters the `resource` context manager, and calls `ensure_bucket()` idempotently. `upload(bucket, key, body, content_type) → str` returns an `s3://` URI. `infrastructure/blob/uri.py`: `raw_key/text_key/report_key` path builders and `to_s3_uri/parse_s3_uri` helpers — all pure functions, no I/O.
`shared/config` — `ScoutSettings` renamed to `IngestSettings`; accessor `settings.ingest`. Added: `rate_limit_rate`, `rate_limit_burst`, `concurrency_per_job` (default 16). `shared/errors` — new leaf types: `FetchError`, `HostRateLimitedError`, `ParseError`, `BlobError` (all under `ExternalServiceError`).
`orchestration/events.py` — wire contracts updated. `IngestPendingEvent.source_ids: list[str]` (fan-out all sources in one message, not one event per source). `IngestCompletedEvent.document_refs: list[IngestDocRef]` and `failed_source_ids: list[str]`. `IngestDocRef` carries `id, source_id, content_hash, blob_uri_raw, char_count`.
`orchestration/workers/scout_worker.py` — new `ScoutWorker(BaseConsumer)`. Consumes `ingest.pending.v1`. Idempotency key: `scout:{job_id}` (one-shot per job, not per attempt). Retry topology: attempt 1 → `retry.ingest.30s`, attempt 2 → `retry.ingest.5m`, attempt 3 → `retry.ingest.1h`, attempt 4+ → `dlq.v1`. On success publishes `IngestCompletedEvent` with OTel `traceparent` injected into headers.
`agents/orchestrator/nodes.py` — `ingest_node` is now dual-mode: if `config["configurable"]["kafka_producer"]` is present (production), publishes `IngestPendingEvent` with all source IDs and returns partial state (stage stays `INGESTING`); if absent (unit tests), falls back to the stub synthetic-doc path so all 164 unit tests continue passing without Docker.
`orchestration/workers/orchestrator_worker.py` — now subscribes to `[Topic.JOBS_REQUESTED, Topic.INGEST_COMPLETED]`. `_handle_job_requested` injects `kafka_producer` into `configurable` and calls `astream(..., interrupt_after=[NODE_INGEST])`. `_handle_ingest_completed` converts `IngestDocRef → DocumentRef`, calls `aupdate_state(config, update, as_node=NODE_INGEST)`, then `astream(None, config)` to resume — the standard LangGraph async-resume pattern (update state as if the interrupted node returned it, then stream from None to continue past the interrupt).
`docker/scout.Dockerfile` — multi-stage, `ingest` + `blob` dep groups only (no `agents`/`llm` — keeps the image lean). `docker-compose.app.yml` — added `scout-worker` service with `minio-init` dependency and `BLOB__*` env vars.
**Tests**: `tests/unit/test_scout_*.py` — pure-function coverage of `content_hash`, `rate_limit` token-bucket math, all four parsers (HTML/RSS/JSON/sitemap), `pipeline.ingest_sources` with full mock tree (happy path, dedup, sitemap expansion, partial failure). `tests/integration/test_scout_e2e.py` — 4 end-to-end tests using `httpx.MockTransport` + real Kafka/Mongo/Redis/MinIO; skips gracefully if `make up` hasn't run.
