## Summary

`shared/metrics/registry.py` ‚Äî `_TrendStormMetrics` declares 30+ Prometheus metrics for all six workers. `_FORBIDDEN_LABELS` frozenset enforces cardinality at import time (job_id/document_id/etc. are trace attributes only, never metric labels). `make_test_metrics()` returns isolated instances for unit tests. `shared/metrics/decorators.py` ‚Äî `@with_metrics` decorator wraps async functions; `_select_labels()` filters to only labels declared on the target metric so one label dict works for any pairing. `shared/metrics/cost.py` ‚Äî `record_llm_cost()` is the single call site for all token accounting; cost computation deferred to Grafana PromQL. `shared/tracing/semantics.py` ‚Äî `Attr` class with ~40 string constants; all span attribute keys in workers MUST use these, never raw string literals. `shared/tracing/__init__.py` extended with `business_span(name, **attrs)` context manager. `infrastructure/metrics/prometheus_server.py` ‚Äî `MetricsServer` runs a WSGI `/metrics` endpoint in a background daemon thread (avoids aiohttp dependency). `api/routers/metrics.py` ‚Äî `GET /metrics` endpoint in the FastAPI app (same Prometheus registry). `infrastructure/kafka/consumer.py` ‚Äî `BaseConsumer` now starts `MetricsServer` in `start()`, stops it in `stop()`, wraps `handle()` via `_timed_handle()` + `_record_handle_metrics()` hook (default no-op; subclasses record worker-specific metrics). `KafkaSettings.metrics_port` (default 9090) drives the port. Three workers override `_record_handle_metrics`: `OrchestratorWorker` ‚Üí `orchestrator_events` counter; `AnalystWorker` ‚Üí `analyst_passes` counter + `analyst_pass_duration` histogram; `SSECoordinatorWorker` ‚Üí `sse_events` counter + `sse_event_duration` histogram. All six workers upgraded to use `Attr.*` constants instead of raw `"trendstorm.*"` string literals in span attributes. `services/retrieval/hybrid.py` upgraded: span attributes use `Attr.*`, RRF stage records `METRICS.analyst_retrieval_hits` per backend (bm25/vector) for funnel width observability. `ops/slo.yml` ‚Äî 13 SLOs. `docker/config/otel-collector.yaml` ‚Äî `tail_sampling` processor added (errors always, latency > 120s always, 5% probabilistic remainder); wired into traces pipeline. `docker/config/prometheus.yml` ‚Äî scrape configs for all 7 services at port 9090. `docker/config/prometheus-alerts.yml` ‚Äî 8 alert rules (DLQ spike, analyst/publisher/API latency SLO, LLM rate limit, Kafka lag, ChromaDB health, Mongo pool) with runbook annotations. `docker/docker-compose.obs.yml` ‚Äî mounts `prometheus-alerts.yml`. `docker/config/grafana/dashboards/trendstorm-overview.json` ‚Äî 6-section overview dashboard (jobs, scout, knowledge, analyst, LLM cost, SSE+publisher, infrastructure). `ops/runbooks/` ‚Äî 8 runbooks (dlq-spike, analyst-latency, publisher-latency, api-latency, llm-rate-limit, kafka-lag, chromadb-health, mongo-pool); each with Signal/Impact/Diagnosis/Remediation/Prevention.
